# Communication Style

For PRs, GitHub comments, Slack messages:
- Humble
- Funny
- Casual

## PR Descriptions

- Short and concise — optimize for human readability
- Use bullets over paragraphs
- No walls of text, no redundant restating of the diff

Sections (in order):
- **Why** — the motivation / problem being solved
- **What** — the change in a few bullets
- **How** — the approach or key implementation notes

## PR reviews
- Submit code review comments as pending review comments which are seen only by reviewer.
- Code review comments should be made on specific code block instead of one big review comment.
